运用，从这里开始...

1.首先，把angular.js引入你的HTML页面中。将script标签放在页面的底部。

2.一切从模块开始 。引入angular.js文件之后会在window下挂一个全局变量叫angular。module表示定义一个模块，指定模块的名称和此模块依赖的其它模块。该函数既可以创建新的模块，也可以获取已有模块，是创建还是获取，通过参数的个数来区分。只有一个参数(模块名),代表获取模块。传递参数不止一个,代表新建模块。空数组代表该模块不依赖其他模块。

像这样：定义了一个模块叫app，有一个依赖模块ui.router。

var app = angular.module('app', ['ui.router']);

像这样：表示获取模块

var getModule = angular.module("app");//app == getModule为true,因为都是同一个模块

注意：一个HTML文档只能注入一个主模块，其他模块通过依赖引用。

3.为模块定义控制器。一个模块可以对应多个控制器。

controller定义一个控制器，第一个参数表示控制器的名称，第二个参数表示控制器的定义。

app.controller('appCtrl',function($scope){//逻辑代码});

你需要为你的控制器指定它的作用范围。就像这样：

<body ng-controller="appCtrl">

控制器的作用范围和设置在的HTML标签范围一致，比如图中范围为整个body。

当实例化一个控制器的时候，会赠送一个$scope对象。$scope默认是个空对象，没有属性，等待你的填充。

这个$scope(也可称为作用域)非常重要，现在就讲讲它的作用。

作用域是用来检测模型的改变和为表达式提供执行上下文的。它是分层组织起来的，并且层级关系是紧跟着DOM的结构的。(这个层级根据的是各个控制器之间的相互嵌套)

我们在$scope上创建我们的逻辑方法和属性。

 $scope.name="myName";
    $scope.handleClick=function(){
         //逻辑代码
    }

注意：外层控制器获取不到内层控制器里设置的属性和方法，内层可以直接获取外层的。如下：

<div ng-controller="a1Controller">
        <p>a1==>message的信息: {{message}}</p>
        <p>age的信息:{{age}}</p>
        <div ng-controller="a2Controller">
            <p>a2==>message的信息:{{message}}</p>
            <p>name的信息: {{name}}</p>
        </div>
    </div>
<script>
            var myApp=angular.module('myApp',[]);
            myApp.controller('a1Controller', function($scope) {
                $scope.name = 'a1里的name';
                $scope.message = 'a1里的message';
            });
            myApp.controller('a2Controller', function($scope) {
                $scope.message = 'a2里的message';
                $scope.age = 'a2里的age';
            });
   </script>

可以看到：我们在两个控制器里都有一个message属性，控制器打印出了各自的message(就近原则)。在a1控制器里还有个name属性，可以在a2控制器中获取到。在a2控制器里的age属性a1却获取不到。不信点这里自己看！

所有的应用都有一个 $rootScope，它可以作用在 ng-app 指令包含的所有HTML元素中。

$rootScope 可作用于整个应用中。是各个 controller 中 scope 的桥梁。用 rootscope 定义的值，可以在各个controller中使用。你可以在它身上设置公用的属性和方法。
