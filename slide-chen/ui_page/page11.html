AngularJS在ie8的兼容处理

1.首先需要确保你的HTML页面开始部分要有DOCTYPE声明。DOCTYPE告诉浏览器使用什么样的HTML或XHTML规范来解析HTML文档。目前的最佳实践就是在HTML文档的首行键入：

<!DOCTYPE html>

DOCTYPE前后有空行也不行，要去掉空格。

2.针对浏览器的内容做标识（使用meta标签调节浏览器的渲染方式）：

bootstrap不支持IE兼容模式，为了让IE浏览器运行最新的渲染模式，将添加以下标签在页面中:

<meta http-equiv="X-UA-Compatible" content="IE=edge,Chrome=1" />

IE=edge表示强制使用IE最新内核，chrome=1表示如果安装了针对IE6/7/8等版本的浏览器插件Google Chrome Frame（可以让用户的浏览器外观依然是IE的菜单和界面，但用户在浏览网页时，实际上使用的是Chrome浏览器内核），那么就用Chrome内核来渲染。

3.版本控制：

使用angular 1.2.29做嵌入式路由、数据交互，1.3.0 以上的版本都不支持ie8。

4.关于路由：

建议使用angular-ui-router 0.2.10 版本，如果把angular-ui-router 换成 0.2.15 的版本就不支持ie8了，所以推测 0.2.10 版本可能用的是 angular 1.3.0 以下的版本写的。

5.关于loader.js

必须引用loader.js，否则无法加载。将angular-loader.js于ui-router后引入。

6.让ie8 能支持bootstrap 3

为了让ie8 能支持bootstrap 3，需要引入3个js文件：html5shiv.js/es5-shim.js/respond.js，要注意，这3个js文件的路径必须要和bootstrap的js 路径保持一致。如下：

<!--[if lt IE 9]>
        <script type="text/javascript" src="bower_components/bootstrap/dist/js/html5shiv.min.js"></script>
        <script type="text/javascript" src="bower_components/bootstrap/dist/js/respond.min.js"></script>
        <script type="text/javascript" src="bower_components/bootstrap/dist/js/es5-sham.min.js"></script>
    <![endif]-->

7.JS与CSS的引入顺序导致的问题：

必须先引用css再引用js

<link rel="stylesheet" type="text/css"  href="bower_components/bootstrap/dist/css/bootstrap.min.css"></script>
    <script type="text/javascript" src="bower_components/bootstrap/dist/js/respond.min.js"></script>
